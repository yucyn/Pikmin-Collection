# V41.1 HOTFIX - 圖片自動判定引擎優化報告

本次 HOTFIX 針對「上傳明信片」的自動標籤判定邏輯進行了全面性的精確化重構，主要解決了各色花朵（特別是藍色與黃色）與蘑菇標籤之間的判定衝突，並嚴格執行區域化掃描。

## 1. 修正與優化內容 (What was fixed/adjusted)

### 核心判定引擎 (`js/imageUpload.js`)
*   **國家判定邏輯優化 (`js/locationParser.js`)**：
    *   **規則重排**：將範圍較小或位於邊界的國家（如韓國、瑞士、加拿大）移至大國（如日本、義大利、美國）之前判定，解決邊境重疊區域的誤判問題。
    *   **座標邊界精確化**：調優了多個國家的 GPS 判定邊界，縮小誤判機率。
    *   **移除重複規則**：清理了代碼中重複的義大利判定規則。
*   **掃描範圍精確化**：
    *   限定只掃描橫向明信片的**右側標籤欄** (x: 68%~100%, y: 0%~100%)。
    *   完全排除左側風景主圖、郵票圖案以外區域以及下方 UI 文字的干擾。
*   **判定優先順序重組**：
    *   **第一優先：花 (Flower)**
        *   採用「貼紙特徵」判定：大型圖案 + 顯著白色描邊/外框。
        *   **色彩中立化**：全面支援紅、黃、藍、白、紫等所有顏色的花朵。
        *   **立即中斷**：一旦確認為花，立即回傳結果，不再進入後續判定。
    *   **第二優先：蘑菇 (Mushroom)**
        *   **嚴格 AND 條件**：必須同時偵測到「星星評分」、「蘑菇名稱文字」以及「CLEARED! 綠色文字」。
        *   支援「銀色/金屬樣式」的薄荷綠文字偵測。
    *   **第三優先：隱藏 (Hidden)**
        *   作為最終預設值，若不符合上述貼紙或蘑菇特徵，一律歸類為隱藏。
*   **版型特殊處理**：
    *   **縱向照片 (Portrait)**：直接判定為隱藏。
    *   **藍色框樣式**：針對具備大面積純藍背景的特殊版型（如 Giant Cheese）進行自動攔截。

## 2. 未調整內容 (What was NOT adjusted)
*   **基礎功能**：圖片上傳流程、GPS 座標讀取、Firebase 儲存邏輯均維持不變。
*   **UI 介面**：未修改上傳按鈕樣式、進度條或任何視覺元件。
*   **其他頁面**：僅針對 `js/imageUpload.js` 進行邏輯修正，不影響其他功能模組。
