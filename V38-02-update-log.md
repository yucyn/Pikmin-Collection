# Pikmin Collection V38 開發者監控儀表板 - 更新日誌

本版本主要導入了全新的「開發者監控牆」系統，並針對現有 UI 進行了精細的質感與佈局優化。

## 1. 開發者即時監控儀表板 (Developer Monitoring Dashboard)
- **實時監控**：新增全站動態追蹤，包含「任務申請」、「留言板訊息」以及「新片上傳」。
- **UID 身份識別**：
    - 所有動態現在都會標註 Firebase Auth UID，方便管理員辨識使用者身份。
    - 修正了留言板系統，現在新訊息會自動記錄發送者的 UID。
- **UID 跨集合搜尋**：
    - 新增搜尋功能，輸入 UID 即可快速檢索該使用者在全站（明信片、任務、留言）的所有活動紀錄。
- **玻璃擬態 UI (Glassmorphism)**：
    - 面板與齒輪按鈕採用高質感的磨砂玻璃特效（Backdrop Blur + Saturate）。
    - 採用精確的動態縮放與位移動畫（Slide-in Animation）。

## 2. 介面佈局優化 (UI/UX Refinement)
- **按鈕整合定位**：
    - 監控按鈕（齒輪）已精確定位於右下角 `bottom: 112px; right: 12px;`。
    - 完美與「回到頂部」及「新增按鈕」垂直排列，互不干擾。
- **UI 精簡化**：
    - 縮小監控面板寬度至 `280px`，齒輪按鈕縮至 `44px`，維持畫面清爽。
- **公告欄優化 (Desktop)**：
    - 電腦版側邊欄公告高度調整為 `250px`。
    - 修正了底部留白 (Padding) 規則，確保長內容在捲動到底部時能完整顯示（預留 `20px` 間距）。
- **全新頁尾資訊 (Site Footer)**：
    - 新增置中對齊的頁尾，包含非官方聲明與聯絡資訊。
    - 針對電腦版側邊欄收合/展開狀態進行位移同步，確保視覺永遠置中。
- **上傳視窗優化 (Upload Modal)**：
    - 更新上傳按鈕下方的提示文字內容。
    - 調整文字為 `0.75em` 精細尺寸，並優化行高與間距。

## 3. 技術修正
- 修正了 CSS 優先級問題，確保側邊欄樣式能正確覆蓋舊有規則。
- 修正了側邊欄收合時，頁尾未同步位移導致視覺偏向右側的問題。
- 優化了監控牆的快照監聽器，確保在開啟/關閉時正確管理資源。
