# GPS / 國家判定測試案例

## 版本
V41.1_HOTFIX_GPS_COUNTRY

---

## 測試目的
確認 Pikbook 的 GPS / country 判定邏輯在新增、編輯、顯示三個流程中都正確。

---

## 測試流程
每一筆資料都要測試三種情境：
1. 新增明信片時自動判定國家
2. 編輯明信片時修改 GPS 後重新判定國家
3. 儲存後重新整理頁面，卡片仍顯示正確國家

---

## 測試案例 1：日本，正確案例
### 地點名稱
Town Hall Pipe Organ
### GPS
35.079833, 136.704991
### 預期國家
日本
### 測試重點
這筆目前可以正確判定為日本。修正後不能被改壞。

---

## 測試案例 2：德國，錯誤案例
### 地點名稱
Ringkirche
### GPS
50.07738, 8.229207
### 地址
Ringkirche, 7, Kaiser-Friedrich-Ring, Rheingauviertel, Wiesbaden, 65197, 德國
### 目前錯誤結果
法國
### 預期國家
德國
### 測試重點
舊資料 country 可能已經是錯誤的「法國」。
修正後只要 GPS 存在，必須重新根據 GPS 判定為「德國」，並覆蓋舊的錯誤 country。

---

## 測試案例 3：奧地利 / 德國邊界附近
### 地點名稱
Gemse Im Schnee
### GPS
47.253184, 10.134661
### 預期結果
請根據 GPS 實際位置判定，不要用標題文字猜。
### 測試重點
這類阿爾卑斯山區地點容易因為地址文字、語言或附近國家造成誤判。

---

## 測試案例 4：德國 Obermaiselstein
### 地點名稱
Sportplatz Obermaiselstein
### GPS
47.443299, 10.242555
### 預期國家
德國
### 測試重點
不得因為附近有奧地利或瑞士地名而判錯。

---

## 測試案例 5：沒有 GPS
### 地點名稱
未知地點
### GPS
空白
### 預期結果
如果沒有 GPS，可以使用使用者手動選擇 country。
如果也沒有 country，顯示「待確認」。

---

## 通過標準
修正後必須通過以下檢查：
- [x] 新增時 GPS 會自動判定 country
- [x] 編輯時 GPS 改變會重新判定 country
- [x] 舊錯誤 country 會被覆蓋
- [x] 沒有 GPS 時不亂猜
- [x] renderCard 只負責顯示，不是唯一修正點
- [x] Firestore 中 country 欄位有被正確更新
- [x] 不影響圖片上傳
- [x] 不影響圖片讀取
- [x] 不影響愛心
- [x] 不影響登入
- [x] 不影響分類
- [x] 不影響搜尋
