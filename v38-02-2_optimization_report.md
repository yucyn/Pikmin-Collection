# Pikmin Collection V38 優化與修復報告

本報告紀錄了本次針對手機版穩定性與佈局所做的核心優化。

## 1. 手機版連線穩定性優化 (js/storage.js)
針對手機網路環境不穩、切換 APP 導致資料中斷的問題進行了深度加強。

- **啟用離線快取 (Offline Persistence)**：開啟 Firestore 離線功能，確保在斷網或不穩時仍能立刻顯示明信片清單。
- **自動重連機制**：當資料監聽器 (Snapshot) 發生錯誤時，系統現在會於 5 秒後自動重試，無需使用者手動刷新。
- **喚醒同步功能**：監聽 `visibilitychange` 事件。當使用者從其他 APP 回到瀏覽器時，系統會自動觸發資料檢查，防止畫面卡死在舊狀態。

## 2. 圖片讀取失敗自動重試 (js/render.js)
解決手機版偶爾發生的「破圖」問題。

- **onerror 重試邏輯**：當明信片圖片載入失敗時，系統會自動在 3 秒後嘗試重新載入一次，並加上時間戳記避開錯誤的快取。

## 3. 蘑菇情報同步穩定性 (js/v37-mushroom-realtime.js)
提升即時情報條的健壯程度。

- **錯誤處理回報**：為蘑菇任務監聽器加入錯誤捕獲與自動重試流程，確保情報條不會因短暫斷線而消失。

## 4. 手機版頁尾佈局修正 (css/style.css, css/v33-sidebar.css)
優化手機版底部的視覺體驗，解決文字與按鈕疊加的問題。

- **極致縮排置中 (Mobile Only)**：針對手機版 (<= 1024px) 進行優化，將頁尾與留言板的左右縮排增加至 **100px**，並限制文字寬度為 **220px**。
- **避開懸浮按鈕**：透過窄版置中佈局，將文字完全鎖定在中央安全區，徹底解決文字與右下角懸浮按鈕（+ 號、回到頂部）重疊的問題。
- **桌機版維持原樣**：相關縮排規則僅作用於手機媒體查詢，桌機版依然保持原本的寬版排列。
- **安全區域適配**：加入 `env(safe-area-inset-bottom)`，確保護蓋在有底線導覽條的裝置上保有正確間距。
